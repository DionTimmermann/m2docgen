Welcome to m2doc

This program converts the header of matlab code files into HTML documents and adds those to the matlab documentation.

Getting Started

To export your own folder with code files to the matlab documentation, copy and modify the file run_m2doc_example.m according to its header. At a minimum, change these options:
  1. toolboxName - A custom name of your toolbox/folder
  2. mFolder - total path to your toobox/folder
  3. outputFolder - total path to the folder where the documentation should be stored
  4. excludeFolder - Exclude the folder name of the documentation (last folder of outputFolder)
  5. opts.toc - if you just want to get started, make sure to leave this empty. The content structure will be mirrored from the subfolder structure.
Make sure that your code headers have a similar structure like the examples in the "Template" folder.

Your own toc stucture

The menu on the left side of the matlab documentation is the "table of content" (toc) structure. By default (if no opts.toc is set), the folder structure will be used as basis for the menue/toc. However, this can be influenced by assigning a special cell to opts.toc. This allows you to merge folders, change folder names and their positions in the toc structure. The cell is defined as follows:

First Column: Heading Names

Each row of the first column contains names, which will be converted to elements in the toc structure.
Custom rule: All undefined Folders will be a found as substructure of the heading of the first toc layer (opts.toc{1,1})

Second Column: Folder Names

Each row contains a string array which defines folders that should be merged into the toc heading of the same row. Example: ["Folder1" "Folder2"] will assign all files whose last folder contains "Folder1" or "Folder2" to the heading in the same row. If left empty, then the heading will be generated, but by default no files will be assigned to it. Set it to "/" or "\" to define the root folder.
Subfolders of a specified folder will also be assigned as a substructure of the new heading.

Third Column: Sub Cells

To define a substructure of a toc heading, you can assign a cell which contains the three columns.

Example

opts.toc = {"Welcome_to_m2doc", "/", {}};
opts.toc{1,3} = {"Dummy", "@Dummy", {};
"MFile", "@MFile", {};
"createDoc", ["@createDoc" "@TemplateHTML"], {}};

Customization

You can customize the resulting HTML documents by modifying the files in the "Templates" folder.

Tips

  • The custom documentation is only visible in the matlab help center if the target export folder is added to the matlab path. m2doc does this step for you, but it can never hurt to check...
  • The conversion of matlab live scripts to html documents is really slow and does not follow the customisation settings, so avoid it when possible.
  • This toolbox heavily relies on "%%" to identify heading sections. However, the code descriptions of this toolbox also contains a lot of "%%", making m2doc ironically a bad target for itself.
  • If you want to define a custom page for one of the table of content (TOC) headings, first define the heading in toc structure in the opts. Then create your custom live script and put it somewhere in the source folder. Make sure that the name of the toc heading and the live script name are compatible with matlabs variable naming scheme. Example: The first heading in the TOC structure is defined as "Welcome_to_m2doc". I created a live script (this one) with the exact same name and put it somewhere in the source folder. If the toc name or file name would include spaces, then the associated file could not be assigned to this landing page.
  • The generation of the search database sometimes fails. Run the script again.
  • Make sure to have m2doc added to the matlab path before running it.
  • Leave option "buildSubDir" on false, as the matlab documentation really only works if all html files are in one folder.
